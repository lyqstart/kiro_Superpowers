# Task Execution Discipline

## 执行 task 前

必须确认：

- 当前 spec 名称；
- 当前 task 编号和标题；
- 对应 requirements / design 部分；
- 预计修改文件；
- 测试/验证命令；
- 完成定义；
- 不做范围。

## 执行 task 中

- 一次只处理一个 task。
- 一个 task 太大时，先拆分。
- 不顺手处理无关问题。
- 每个行为变化都要有测试或明确验证。
- 遇到不清楚的设计/需求，停止并询问。

## 执行 task 后

必须：

- 运行验证命令；
- 读取完整输出；
- 做规格符合性审查；
- 做代码质量审查；
- 报告未解决风险；
- 只有验证通过才标记完成。
