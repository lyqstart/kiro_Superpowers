# Verification Before Completion

## 核心规则

没有新鲜验证证据，不允许说完成。

## 完成前必须执行

1. 指出证明当前结论的命令。
2. 运行完整命令。
3. 读取输出和退出码。
4. 汇报实际结果。
5. 再判断是否完成。

## 不能接受的说法

- 应该可以；
- 看起来没问题；
- 我认为已经修好；
- 之前跑过；
- agent 报告成功；
- lint 通过所以 build 应该也通过。

## 报告格式

```text
验证命令：...
退出状态：...
通过：...
失败：...
关键输出：...
结论：通过 / 未通过 / 阻塞
```
